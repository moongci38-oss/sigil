## 1. 테스트 계층 및 비율

| 계층 | 비율 목표 | 실행 시점 | 도구 | 대상 |
|------|:-------:|---------|------|------|
| 단위 테스트 (Unit) | ~70% | 매 커밋 (pre-push) | | 함수, 클래스, 컴포넌트 |
| 통합 테스트 (Integration) | ~20% | 매 커밋 (CI) | | API 엔드포인트, DB 연동 |
| E2E 테스트 | ~10% | staging 배포 후 | | 핵심 유저 플로우 |
| 모바일 디바이스 E2E | (E2E 내) | staging 배포 후 | Playwright devices | 주요 모바일 시나리오 |

> 비율은 테스트 피라미드 원칙 기반. 프로젝트 특성에 맞게 조정한다.

## 2. 프론트엔드 테스트 전략

| 대상 | 테스트 방식 | 도구 | 예시 |
|------|-----------|------|------|
| UI 컴포넌트 | 단위 | | 렌더링 + 사용자 이벤트 |
| 커스텀 훅 | 단위 | | 상태 변화 + 반환값 |
| 페이지 통합 | 통합 | | 라우팅 + API 호출 |
| API 모킹 | 통합 | | 네트워크 요청 가로채기 |
| 접근성 | 자동 | | ARIA 규칙 위반 탐지 |
| 반응형/모바일 렌더링 | 단위/통합 | | `matchMedia` 모킹으로 breakpoint별 검증 |
| 비주얼 리그레션 | 선택 | | 스크린샷 비교 (데스크톱 + 모바일 뷰포트) |

### FE 테스트 작성 원칙
- 구현 디테일이 아닌 **사용자 행동** 기반 테스트
- `data-testid`를 사용하되 ARIA role/label 우선
- 스냅샷 테스트는 최소화 (변경에 취약)

## 3. 백엔드 테스트 전략

| 대상 | 테스트 방식 | 도구 | 예시 |
|------|-----------|------|------|
| Service 비즈니스 로직 | 단위 | | 입력/출력/예외 |
| API 엔드포인트 | 통합 | | HTTP 요청/응답/상태코드 |
| Guard/Middleware | 단위 | | 인증/인가 시나리오 |
| DB 연동 (Repository) | 통합 | | CRUD + 트랜잭션 |
| 외부 API 연동 | 단위 | | Mock 기반 에러 핸들링 |

### BE 테스트 작성 원칙
- 서비스 레이어: 의존성 Mock 주입
- 컨트롤러 레이어: 실제 HTTP 요청/응답 검증 (supertest 등)
- DB 레이어: 테스트 DB 사용 (인메모리 or 컨테이너)

## 6. 테스트 환경 설정

| 환경 | 파일/설정 | 용도 |
|------|---------|------|
| 환경변수 | `.env.test` | 테스트 전용 DB/API URL |
| 테스트 러너 설정 | {jest.config / vitest.config} | 모듈 매핑, 타임아웃, 커버리지 |
| CI 설정 | {GitHub Actions / etc} | 테스트 실행 파이프라인 |
| E2E 설정 | {playwright.config} | 브라우저, 뷰포트, 타임아웃 |

### CI 파이프라인 테스트 흐름

```
PR 생성/업데이트
  → lint + type-check
  → unit tests (FE + BE)
  → integration tests (API)
  → build
  → (staging 배포 시) E2E tests
```

## 9. 성능 테스트 기준

### 9.1 Core Web Vitals 목표
| 지표 | 목표값 | 측정 시점 | Pass 기준 |
|------|:------:|---------|---------|
| LCP | < 2.5s | Lighthouse CI | P75 기준 |
| FID/INP | < 100ms/200ms | Web Vitals | P75 기준 |
| CLS | < 0.1 | Lighthouse CI | P75 기준 |
| TTFB | < 800ms | 서버 모니터링 | P95 기준 |

### 9.2 번들 사이즈 모니터링
| 체크 대상 | 방법 | CI 연동 |
|----------|------|:------:|
| First Load JS | `next build` 분석 | PR 코멘트 |
| 라우트별 JS | Bundle Analyzer | 수동 점검 |
| 이미지 최적화 | Lighthouse audit | CI |

### 9.3 성능 리그레션 방지
| 전략 | 설명 |
|------|------|
| Lighthouse CI | PR별 자동 실행, 점수 하락 시 경고 |
| Bundle Size Check | PR별 번들 사이즈 변화량 코멘트 |
| Core Web Vitals Alert | 프로덕션 모니터링, 임계치 초과 시 알림 |

> 이 섹션은 **목표값 설정 수준**이다. 구체적 구현(Lighthouse CI 설정 파일, 모니터링 코드)은 Trine에서 다룬다.
