## 6. 반응형/적응형 설계

### 6.1 브레이크포인트

| 디바이스 | 너비 | 방향 | 레이아웃 변경 |
|---------|------|:----:|-------------|
| 모바일 (세로) | ~480px | Portrait | 단일 컬럼, Bottom Nav, 카드 풀와이드 |
| 모바일 (가로) | ~812px | Landscape | 2컬럼 가능, 네비게이션 유지 |
| 태블릿 | 481~1024px | Both | 2컬럼 그리드, 사이드바 접힘 가능 |
| 데스크톱 | 1025px~ | Landscape | 멀티 컬럼, 사이드바 펼침 |

### 6.2 적응형 요소

| 요소 | 모바일 | 태블릿 | 데스크톱 |
|------|--------|--------|---------|
| 네비게이션 | Bottom Nav / Hamburger | 축소 사이드바 / 탭 | 펼친 사이드바 / 헤더 |
| 데이터 테이블 | 카드 리스트로 변환 | 수평 스크롤 | 풀 테이블 |
| 모달/다이얼로그 | Full-screen / Bottom Sheet | 센터 모달 (중간) | 센터 모달 (큰) |
| 폼 레이아웃 | 단일 컬럼 | 2컬럼 (관련 필드 그룹핑) | 2~3컬럼 |
| 이미지 갤러리 | 1컬럼 풀와이드 | 2컬럼 그리드 | 3~4컬럼 그리드 |
| 사이드 패널 | 숨김 (토글) | 오버레이 | 인라인 |

### 6.3 모바일 우선 설계 원칙

> **Mobile-first가 기본**이다. Desktop-only 페이지는 명시적으로 선언해야 한다.

| 대상 | 기본 정책 | 예외 허용 조건 |
|------|----------|-------------|
| **서비스 (사용자 앱/웹)** | Mobile-first | 없음 — 모바일 대응 필수 |
| **관리자 대시보드** | Mobile-first | 복잡한 데이터 테이블 화면만 Desktop-only 선언 가능 |
| **운영 도구** | Mobile-first | 특수 입력 장치 필요 화면만 Desktop-only 선언 가능 |

**Desktop-only 선언 형식** (예외 화면에만 적용):
```
| 화면 | Desktop-only 사유 |
|------|-----------------|
| {화면명} | {사유: 예 — 복잡한 Gantt 차트, 드래그앤드롭 에디터 등} |
```

### 6.4 Safe Area 대응

> iOS notch/Dynamic Island/Home Indicator와 Android system bars에 대한 안전 영역을 확보한다.

| 영역 | 대응 방법 | CSS |
|------|---------|-----|
| iOS 상단 (Notch/Dynamic Island) | 콘텐츠가 노치 영역에 겹치지 않도록 padding | `padding-top: env(safe-area-inset-top)` |
| iOS 하단 (Home Indicator) | 하단 고정 요소(Bottom Nav, CTA)에 여백 확보 | `padding-bottom: env(safe-area-inset-bottom)` |
| iOS 좌우 (가로 모드 노치) | 가로 모드에서 좌우 콘텐츠 잘림 방지 | `padding-left: env(safe-area-inset-left)` |
| Android 시스템 바 | 상태바/네비게이션 바 영역 확보 | `viewport-fit=cover` + safe-area env |

**viewport 메타 태그** (필수):
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">
```

### 6.5 모바일 폼 UX 가이드

> 모바일에서 폼 입력은 사용성에 가장 큰 영향을 미친다. 키보드 타입, 선택 컴포넌트, 피드백 위치를 명확히 정의한다.

#### 키보드 타입 매핑

| 입력 유형 | `inputMode` | `type` | iOS 키보드 | 비고 |
|---------|------------|--------|-----------|------|
| 일반 텍스트 | `text` | `text` | 기본 키보드 | |
| 이메일 | `email` | `email` | @ 포함 키보드 | |
| 전화번호 | `tel` | `tel` | 숫자 패드 | |
| 숫자 (정수) | `numeric` | `text` | 숫자 패드 | `type="number"` 대신 `inputMode` 사용 권장 |
| 금액 | `decimal` | `text` | 소수점 포함 숫자 | |
| URL | `url` | `url` | .com / / 포함 키보드 | |
| 검색 | `search` | `search` | 검색 버튼 (Go 대신) | |
| 비밀번호 | — | `password` | 기본 + 자동완성 | `autocomplete` 속성 병행 |

#### 날짜/시간 선택기

| 입력 유형 | 모바일 권장 | 데스크톱 권장 | 비고 |
|---------|-----------|------------|------|
| 날짜 | 네이티브 `<input type="date">` | 캘린더 피커 | 네이티브가 모바일 UX 최적 |
| 시간 | 네이티브 `<input type="time">` | 드롭다운/피커 | |
| 날짜+시간 | 네이티브 `<input type="datetime-local">` | 복합 피커 | |
| 날짜 범위 | 2개 date input (시작/종료) | 범위 캘린더 | |

#### 선택 컴포넌트

| 옵션 수 | 모바일 권장 | 데스크톱 권장 |
|:-------:|-----------|------------|
| 2~3개 | 라디오 버튼 / 세그먼트 컨트롤 | 라디오 / 세그먼트 |
| 4~7개 | 칩 그룹 / 바텀시트 셀렉트 | 드롭다운 |
| 8개+ | 검색 가능 바텀시트 | 검색 드롭다운 |
| Yes/No | 토글 스위치 | 체크박스 / 토글 |

#### 입력 유효성 피드백 위치

| 피드백 유형 | 위치 | 타이밍 |
|-----------|------|--------|
| 필드별 에러 | 입력 필드 **바로 아래** (인라인) | blur 시 또는 입력 중 (debounced) |
| 폼 전체 에러 | 폼 **상단** (스크롤 위치 자동 이동) | 제출 시 |
| 성공 피드백 | 토스트 알림 (상단 또는 하단) | 제출 완료 시 |
| 진행 표시 | 제출 버튼 내 스피너 교체 | 제출 중 |

> **모바일 폼 추가 규칙**:
> - 키보드가 올라올 때 활성 필드가 가려지지 않도록 스크롤 조정
> - 멀티스텝 폼은 프로그레스 바 표시
> - 필수 필드 표시: `*` 또는 "(필수)" 레이블
> - 자동완성(`autocomplete`) 속성 적극 활용 (이름, 주소, 카드번호)
