## 1. 디자인 원칙

### 1.1 핵심 원칙

| 원칙 | 설명 | 적용 예시 |
|------|------|---------|
| **데이터 밀도 우선 (Data Density)** | 한 화면에 최대한 많은 정보를 정돈하여 표시 | 대시보드: 6개 KPI 카드 + 2개 차트 + 2개 테이블 동시 노출 |
| **효율적 워크플로우 (Minimal Clicks)** | 주요 작업 3클릭 이내 완료 | 유저 조회 → 제재: 검색(1) → 유저 선택(2) → 제재 액션(3) |
| **명확한 계층 구조 (Clear Hierarchy)** | 사이드바 > 메인 콘텐츠 > 디테일 패널 | 좌측 네비게이션 + 중앙 테이블 + 우측 사이드패널 |
| **안전한 조작 (Safe Actions)** | 되돌릴 수 없는 액션에 이중 확인 | 영구 밴, 피처 플래그 변경, 코인 지급 → 확인 모달 필수 |
| **실시간 인지 (Real-time Awareness)** | 핵심 지표의 실시간 갱신 + 변화 알림 | CCU, 에러율 5초 갱신, 임계치 초과 시 알림 뱃지 |

### 1.2 디자인 키워드

- **Clean** — 깔끔하고 정돈된 레이아웃, 불필요한 장식 배제
- **Professional** — 신뢰감 있는 중성적 색상, 일관된 타이포그래피
- **Efficient** — 최소 클릭으로 목표 달성, 벌크 액션 지원
- **Informative** — 차트/그래프로 데이터 시각화, 트렌드 표시

### 1.3 접근성 목표

- WCAG AA 준수
- 최소 클릭 타겟: 36x36px (데스크톱 기준)
- 색상 대비 비율: 4.5:1+ (텍스트)
- 키보드 네비게이션 완전 지원 (Tab, Enter, Esc)
- 라이트/다크 모드 토글 지원

## 3. 컴포넌트 스펙

### 3.1 사이드바 네비게이션

```
┌─────────────────┐
│ [로고] 바둑이 Admin │
├─────────────────┤
│ ▶ 대시보드        │  ← 활성: 배경 Primary/10%, 텍스트 Primary
│   유저 관리       │
│   게임 관리       │
│   경제 관리       │
│   콘텐츠 관리     │
│   시스템          │
│                   │
│                   │
│                   │
├─────────────────┤
│ [아바타] Admin     │  ← 하단 고정
│ Super Admin       │
│ [로그아웃]         │
└─────────────────┘
```

| 속성 | 값 |
|------|:--|
| 너비 | 펼침: 240px, 접힘: 64px |
| 배경 | `--admin-surface` |
| 활성 항목 | 배경 `--admin-primary` 10% + 텍스트 `--admin-primary` + 좌측 3px 인디케이터 |
| 비활성 항목 | 텍스트 `--admin-text-secondary` |
| 호버 | 배경 `--admin-surface-hover` |
| 아이콘 | 24x24px, 좌측 고정 |
| 접기 토글 | 하단 chevron 아이콘 버튼 |
| 하위 메뉴 | 들여쓰기 16px + 작은 글씨 |

### 3.2 데이터 테이블

| 속성 | 값 |
|------|:--|
| 행 높이 | 48px |
| 헤더 | 배경 `--admin-bg`, 텍스트 `--admin-text-secondary` Bold |
| 행 호버 | 배경 `--admin-surface-hover` |
| 행 선택 | 배경 `--admin-primary` 5% + 체크박스 활성 |
| 정렬 | 헤더 클릭 → ASC/DESC 토글 (화살표 아이콘) |
| 페이지네이션 | 하단: "1-20 of 342" + 이전/다음 + 페이지 크기 선택 (20/50/100) |
| 필터 | 상단 필터 바: 검색 + 드롭다운 필터 (상태, 기간) |
| 벌크 액션 | 체크박스 2개+ 선택 시 상단 액션 바 노출 ("N개 선택됨 | [제재] [내보내기]") |
| 빈 상태 | 중앙 아이콘 + "데이터가 없습니다" + 필터 초기화 버튼 |

### 3.3 KPI 카드

```
┌──────────────────────┐
│  현재 접속자 (CCU)     │  ← Caption, --admin-text-secondary
│  1,234                │  ← H1, --admin-text
│  ▲ +12% (어제 대비)    │  ← 상승: 녹색, 하락: 빨강
│  ━━━━━━━━━━━━━━━━━━  │  ← 미니 스파크라인 (7일 추세)
└──────────────────────┘
```

| 속성 | 값 |
|------|:--|
| 크기 | 최소 200px 너비, 고정 높이 120px |
| 배경 | `--admin-surface` |
| 그림자 | `--admin-shadow-sm` |
| 모서리 | `--admin-radius-md` |
| 상승 지표 | ▲ `--admin-success` |
| 하락 지표 | ▼ `--admin-error` |
| 스파크라인 | 7일 추세 미니 라인 차트 (높이 24px) |

### 3.4 차트 컴포넌트

| 차트 유형 | 용도 | 라이브러리 |
|---------|------|---------|
| 라인 차트 (시계열) | DAU/MAU/매출 추세 | Chart.js 또는 Recharts |
| 바 차트 | 모드별/기간별 비교 | Chart.js 또는 Recharts |
| 도넛/파이 차트 | 수익원 비율, 유저 분포 | Chart.js 또는 Recharts |
| 히트맵 | 시간대별 접속 패턴 | D3.js |
| 스파크라인 | KPI 카드 내 미니 추세 | Chart.js (미니 모드) |

**공통 속성**:
- 툴팁: 마우스 호버 시 상세 데이터
- 범례: 차트 하단 또는 우측
- 기간 선택: 7일 / 30일 / 90일 / 커스텀 (DateRangePicker)
- 빈 상태: "데이터를 로딩 중입니다" 또는 "선택한 기간에 데이터가 없습니다"

### 3.5 필터 패널

```
┌──────────────────────────────────────────────────┐
│ [검색: UID/닉네임/이메일]  [상태: 전체 ▼]  [기간: 7일 ▼]  [필터 초기화] │
└──────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|:--|
| 검색 필드 | 실시간 debounce (300ms) + 돋보기 아이콘 |
| 드롭다운 필터 | 단일/다중 선택, 체크박스 리스트 |
| 기간 선택 | 프리셋 (오늘/7일/30일/90일) + 커스텀 DateRangePicker |
| 필터 초기화 | Ghost 버튼 "초기화" |
| 활성 필터 | 필터 바 하단에 태그 뱃지로 표시 (X로 제거 가능) |

### 3.6 피처 플래그 토글

```
┌──────────────────────────────────────────────────┐
│  FEATURE_TAN_SYSTEM                               │
│  설명: 탄 시스템 전체 ON/OFF (심의 핵심)             │
│  상태: [■ OFF]   ← 토글 스위치 (빨강: OFF, 초록: ON) │
│  대상: 전체 유저                                    │
│  마지막 변경: 2026-02-20 14:30 by super_admin       │
│                                                    │
│  ⚠️ 경고: 이 플래그 변경 시 탄 시스템이 전체 활성화됩니다. │
│  심의 빌드에서는 반드시 OFF를 유지하세요.               │
│                                                    │
│  [변경 이력 보기]                                   │
└──────────────────────────────────────────────────┘
```

| 속성 | 값 |
|------|:--|
| 토글 크기 | 44x24px |
| ON 색상 | `--admin-success` |
| OFF 색상 | `--admin-error` |
| 변경 시 | 확인 모달 필수 ("정말 [플래그명]을 [ON/OFF]하시겠습니까?") |
| 대상 지정 | 전체 / 비율(%) / 특정 유저 그룹 |
| 변경 로그 | 누가, 언제, 이전값→새값 기록 |

## 5. 인터랙션 패턴

### 5.1 벌크 액션

| 단계 | 동작 | 피드백 |
|:---:|------|--------|
| 1 | 체크박스로 항목 선택 (2개+) | 상단에 "N개 선택됨" 액션 바 슬라이드 다운 |
| 2 | 벌크 액션 버튼 클릭 (예: "일괄 경고") | 확인 모달: "N명에게 경고를 보내시겠습니까?" |
| 3 | 확인 | 처리 중 스피너 → 완료 토스트 "N명 경고 완료" |
| 4 | 취소 | 모달 닫힘, 선택 유지 |

### 5.2 검색 + 필터

| 동작 | 방식 | 피드백 |
|------|------|--------|
| 검색 입력 | debounce 300ms → API 호출 | 테이블 즉시 업데이트 + 결과 수 표시 |
| 필터 선택 | 드롭다운 선택 → 즉시 적용 | 활성 필터를 태그 뱃지로 표시 |
| 필터 제거 | 태그 뱃지 X 클릭 | 해당 필터 제거 + 테이블 갱신 |
| 전체 초기화 | "초기화" Ghost 버튼 | 모든 필터/검색어 제거 |

### 5.3 CRUD 워크플로우 (예: 공지사항)

| 동작 | UI | 피드백 |
|------|:--|--------|
| Create | "새 공지 작성" → 폼 모달 | 모달: 제목/유형/내용/게시일 입력 → "저장" → 토스트 "공지 등록 완료" |
| Read | 테이블 행 클릭 → 사이드패널 | 사이드패널: 상세 정보 표시 |
| Update | 사이드패널 "편집" → 폼 모달 | 기존값 프리필 → 수정 → "저장" → 토스트 "수정 완료" |
| Delete | 사이드패널 "삭제" → 확인 모달 | "이 공지를 삭제하시겠습니까?" → "삭제" → 토스트 "삭제 완료" |

### 5.4 피처 플래그 변경

| 단계 | 동작 | 피드백 |
|:---:|------|--------|
| 1 | 토글 클릭 | 확인 모달: "FEATURE_X를 [ON/OFF]하시겠습니까?" + 영향 설명 |
| 2 | 모달에서 확인 | API 호출 → 로딩 → 토스트 "플래그 변경 완료" |
| 3 | 로그 기록 | 자동: 변경자, 이전값, 새값, 타임스탬프 기록 |
| 4 | 즉시 반영 | Firebase Remote Config 업데이트 → 클라이언트 반영 (재시작 불필요) |

### 5.5 실시간 데이터 갱신
