# 리서치 방법론

## 구조화된 리서치 접근

### 1. 경쟁 가설 개발
- 조사 주제에 대해 최소 2개 이상의 경쟁 가설을 수립한다
- 가설별로 검증에 필요한 데이터 소스를 사전 식별한다
- 가설 트리를 활용하여 논리 구조를 시각화한다

### 2. 출처 다중 검증
- 단일 소스에 의존하지 않는다
- 검색 도구 활용 순서: Brave Search (실시간) → Context7 (기술 문서) → 도메인 전문 사이트
- 1차 소스(공식 문서, 원논문)를 우선한다

### 3. 신뢰도 등급

모든 리서치 결과에 신뢰도를 표기한다:

| 등급 | 기준 | 표기 |
|------|------|------|
| **High** | 다중 소스에서 일관되게 확인 | `[신뢰도: High]` |
| **Medium** | 단일 신뢰 소스 또는 2개 소스 부분 일치 | `[신뢰도: Medium]` |
| **Low** | 단일 비공식 소스 또는 AI 추정 | `[신뢰도: Low]` |

### 4. 조사 노트 활용
- 장기 리서치는 `01-research/` 하위에 조사 노트를 작성하며 진행
- 중간 발견 사항, 미해결 질문, 추가 조사 필요 항목을 기록
- 최종 보고서와 조사 노트를 분리하여 관리

## 리서치 결과 형식

### 필수 포함 항목
- **출처**: URL + 접근 날짜
- **신뢰도 등급**: High / Medium / Low
- **데이터 시점**: 자료의 기준 시점 (예: 2026년 1월 기준)
- **제한사항**: 알려진 편향, 표본 크기, 지역 제한 등

### 수치 데이터 보고
- 범위 제시 우선 (예: "시장 규모 $50-70B" vs "$60B")
- 정확한 수치는 소스 직접 인용
- 예측/전망은 반드시 출처 기관 + 발표일 명시
