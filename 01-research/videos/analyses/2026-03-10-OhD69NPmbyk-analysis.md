## 4. 비판적 분석

### 주장 1: "Claude Projects의 지속적 컨텍스트로 Zero-context 명령이 가능하다"

- **근거**: 프로젝트 지식에 문서를 적재하면 이후 대화에서 해당 컨텍스트를 자동 참조한다고 시연
- **근거 유형**: 경험적 시연 (데모)
- **한계**: Projects 기능의 컨텍스트 창 크기 제한, 지식 업데이트 시점, 지침 문서 수 상한선 등 기술적 제약을 언급하지 않음. "아무런 맥락을 주지 않아도"는 과장 — 실제로는 지식 문서가 컨텍스트로 주입됨.
- **반론/대안**: 긴 프로젝트일수록 오래된 지식 문서가 최신 대화를 오염시킬 수 있음. 주기적인 지식 관리 전략이 필요하나 언급 없음.

### 주장 2: "AI 생성 글도 제대로 작성하면 검색 상위 노출된다"

- **근거**: 채널 운영자가 ChatGPT Canvas 및 Claude로 작성한 블로그 글이 '주노헤어 싱가포르' 키워드에서 상위 노출됨을 스크린샷으로 제시
- **근거 유형**: 경험적 (단일 사례)
- **한계**: 단일 사례(저경쟁 롱테일 키워드)를 일반화함. 네이버 검색 알고리즘 변화, 경쟁 키워드에서의 결과는 다를 수 있음. 장기 지속 여부 미검증.
- **반론/대안**: 구글/네이버 공식 가이드라인에서는 AI 생성 콘텐츠 자체를 제재하지 않되, E-E-A-T(경험·전문성·권위·신뢰)를 충족해야 한다고 명시. 인간의 편집·검수 없이 완전 자동화된 대량 발행은 여전히 위험 요인.

### 주장 3: "글쓰기 도구로는 Claude가 GPT·Perplexity 등보다 우위"

- **근거**: "많은 AI를 긴 시간 경험하면서 느낀 것"이라는 개인 경험 진술
- **근거 유형**: 의견/경험
- **한계**: 비교 기준이나 평가 방법론 없음. 주관적 선호일 가능성 높음.
- **반론/대안**: 글쓰기 성능은 작업 유형(창작 vs. 기술 문서 vs. 번역)에 따라 달라짐. GPT-4o의 Long-form Writing 성능, Gemini의 다국어 처리 등 경쟁 우위 영역이 존재함.

### 주장 4: "기획 제안 시 상대방이 원하는 것을 먼저 파악해야 한다"

- **근거**: 입사 지원 시 회사 인재상 파악 비유
- **근거 유형**: 의견/유추
- **한계**: 당연한 원칙이지만 Claude가 '상대방 니즈 분석'을 얼마나 정확하게 수행하는지 검증 없음.
- **반론/대안**: 맥락 없이 Claude에게 "백화점이 중요하게 생각하는 것"을 물으면 일반적 답변이 나올 수 있음. 실제 바이어 인터뷰(Mom Test)나 1차 데이터 없이 AI 추정에만 의존하는 것은 한계.

## 5. 팩트체크 대상

## 6. 실행 가능 항목

- [ ] **Claude Projects 지식 구조 설계 실험** — 비즈니스 워크스페이스에서 SIGIL 프로젝트별 Projects 생성. 각 Projects에 S4 산출물(상세 기획서, 개발 계획)을 지식으로 적재하여 반복 문서 생성 효율 측정. 적용 대상: BUSINESS 비개발 업무 (기획서, 이메일)

- [ ] **2-step 프롬프트 워크플로우 표준화** — 외부 문서 작성 전 "상대방이 중요하게 생각하는 요소 먼저 물어보기 → 초안 작성" 순서를 `.claude/skills/` 내 비즈니스 커뮤니케이션 스킬에 통합. 적용 대상: 파트너십 이메일, 외부 제안서

- [ ] **콘텐츠 작성 지침 파일 생성** — `04-content/templates/blog-review-guideline.md` 생성: 검색 상위 노출 + 자연스러운 리뷰글 요건 정리. 향후 Claude Projects 지식으로 재사용. 적용 대상: 콘텐츠 마케팅 작업 시

- [ ] **한/영 동시 출력 프롬프트 템플릿 등록** — 글로벌 커뮤니케이션 시 표준 프롬프트("한국어와 영어로 각각 작성해줘")를 스킬 템플릿화. 적용 대상: 북미/글로벌 시장 대상 커뮤니케이션

- [ ] **AI 글쓰기 SEO 소규모 실험** — Claude로 생성한 블로그 글 3-5개를 롱테일 키워드로 네이버 발행 후 4주간 순위 변화 트래킹. `01-research/` 실험 결과 기록. 적용 대상: 04-content/ 콘텐츠 전략 검증
