## 4. 비판적 분석

### 주장 1: "ClawJack 체크섬 자가 치유로 공급망 공격을 방어한다"
- **근거**: 해시 불일치 시 신뢰 저장소에서 자동 재다운로드하는 메커니즘 설명 및 CI/CD 연동 언급
- **근거 유형**: 기술 구현 설명 (경험적)
- **한계**: "신뢰 저장소" 자체가 침해되었을 경우의 대응 방안 미언급. 검증 주기(실시간/주기적)와 신뢰 저장소의 서명 체계 상세 불명확.
- **반론/대안**: 체크섬 검증은 필요조건이지 충분조건이 아니다. 신뢰 저장소에 GPG 서명 등 별도 인증 체계가 없으면 한 계층 수준의 방어에 그친다.

### 주장 2: "결정론적 워크플로우가 전체 에이전트 실행(full agentic)보다 오류 찾기가 쉽다"
- **근거**: 각 에이전트가 고정된 순서로 실행되고 전담 검증자가 각 단계를 검증하는 구조 설명
- **근거 유형**: 설계 원칙 설명 (의견)
- **한계**: 결정론적 워크플로우는 예측 불가능한 입력 변화에 취약하다. 복잡한 비선형 작업에서의 성능 비교 데이터 없음.
- **반론/대안**: 결정론적 설계는 유연성을 포기하고 예측 가능성을 얻는 트레이드오프다. Trine 파이프라인처럼 이미 단계적 구조를 갖춘 시스템에서는 중복될 수 있다.

### 주장 3: "LanceDB 플러그인이 OpenCode 내장 메모리를 실질적으로 대체한다"
- **근거**: 라이브러리 선호도를 저장하고 바이너리 파일에서 정확히 복원하는 단순 시연
- **근거 유형**: 단순 기능 검증 시연 (경험적, 범위 제한적)
- **한계**: 단일 선호도 저장·복원이라는 극히 단순한 시나리오만 검증됨. 수천 개 메모리 항목에서의 재정렬 품질, 레이턴시, 정밀도(precision) 등 실제 운영 성능 미검증.
- **반론/대안**: 메모리 시스템의 효과는 회상 정확도(recall@K)로 검증해야 한다. "올바르게 구성됨" 확인 수준에 그쳐 실제 품질 개선 폭을 판단하기 어렵다.

### 주장 4: "Unbrowse는 Playwright보다 보안·성능이 우수하다"
- **근거**: "캡처 코드가 로컬에 남고 아무것도 시스템을 떠나지 않는다", 세션 유지 가능
- **근거 유형**: 기술 특성 설명 (의견)
- **한계**: API 역엔지니어링 자체가 서비스 이용약관(ToS) 위반 가능성 내포. 브라우저 쿠키 직접 취급 시 쿠키 탈취 위험 존재. 영상 내 "에이전트가 네트워크 부분에 접근하는 것은 보안 위험"이라는 자체 인정 발언이 있음.
- **반론/대안**: "로컬 실행 = 안전"은 불완전한 등식이다. 역엔지니어링 기반 방식은 대상 사이트 API 변경에 취약하고 법적 리스크가 존재한다.

### 주장 5: "MoltWorker로 서버 관리 없이 OpenCode를 프로덕션에 운영할 수 있다"
- **근거**: Cloudflare Workers 기반 배포 구조, 다채널 지원, AI Gateway 모델 전환 기능 설명
- **근거 유형**: 기능 설명 (의견)
- **한계**: 영상 제작자 본인이 "현재 실험적(experimental)이며 프로세스 인수에 시크릿이 노출되는 보안 문제"가 있다고 명시함.
- **반론/대안**: 보안 이슈를 스스로 인정한 실험적 도구를 프로덕션에 사용하는 것은 시기상조다. 개인 프로젝트 수준의 탐색 용도로만 적합하다.

## 5. 팩트체크 대상

- **주장**: "Cisco가 OpenCode를 완전한 보안 위험(security hole)으로 공식 지적했다." | **검증 필요 이유**: Cisco의 실제 보안 리포트 원문 존재 여부와 대상 프레임워크의 정확한 이름 확인 필요. 자막 오인식("항공"은 "hole"의 오전사 추정)으로 원문 표현 불명확. | **검증 방법**: Cisco Talos 또는 Cisco Security Advisories에서 오픈소스 에이전트 프레임워크 관련 보고서 검색.

- **주장**: "Prompt Security는 SentinelOne 자회사이다." | **검증 필요 이유**: 자막은 "센터넬일 자회사"로 전사. Prompt Security와 SentinelOne의 실제 관계(인수/자회사/파트너십) 불명확. | **검증 방법**: Prompt Security 공식 사이트 및 LinkedIn에서 회사 관계 확인. SentinelOne 공식 인수 발표 검색.

- **주장**: "Jina 임베딩 모델은 일정 토큰까지 무료다." | **검증 필요 이유**: 자막에 "최대만 토큰"으로 전사되어 정확한 수치 불명확. Jina의 현재 요금제는 변경될 수 있다. | **검증 방법**: Jina AI 공식 사이트(jina.ai) 요금제 페이지에서 현재 무료 티어 한도 확인. LanceDB 플러그인 README에서 기본 임베딩 모델 설정 확인.

## 7. 시스템 적용 맥락

| 영상 제안 | 현재 상태 | 갭 | 우선순위 |
|----------|----------|-----|---------|
| 스킬 체크섬 무결성 검증 (ClawJack) | `.claude/skills/` 수동 관리, 별도 CVE 스캔 없음 | 공급망 공격 방어 체계 부재 | 중 (보안) |
| 결정론적 에이전트 + 전담 검증자 (Entfarm) | Trine: Wave 기반 Subagent 순차 스폰, Check 3.5/3.7이 검증자 역할 수행 | 개념적으로 이미 유사 구현됨. YAML 에이전트 프롬프트 방식은 추가 조사 가치 있음 | 낮 (이미 유사 구현) |
| 하이브리드 벡터 메모리 + re-ranking (LanceDB) | session-state.mjs + MEMORY.md 기반 텍스트 메모리 | 벡터 검색 기반 관련성 우선 회상 없음. 세션 축적 시 검색 품질 저하 가능 | 중 (장기 운영) |
