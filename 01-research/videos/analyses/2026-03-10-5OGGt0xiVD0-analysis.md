## 4. 비판적 분석

### 주장 1: "AGENTS.md는 코딩 에이전트 성능을 저하시킨다"

- **주장**: 컨텍스트 파일을 제공하면 에이전트 성능이 오히려 하락한다.
- **제시된 근거**: 학술 벤치마크 연구 (모델명 일부 언급: Claude 3.5, GPT 계열, Gemini 3), 자체 실험 (1분 11초 vs 1분 29초).
- **근거 유형**: 실증(연구 인용) + 경험(단일 사례 실험)
- **한계**: 인용한 연구의 출처(논문명, 저자)를 명시하지 않아 독립 검증이 불가능. 자체 실험은 1회성이며 화자 본인도 인정. 테스트된 태스크 유형(GitHub 이슈 해결)이 모든 코딩 태스크를 대표하지 않을 수 있음.
- **반론/대안**: 도메인 전문 지식(내부 API, 독점 라이브러리, 특수 컨벤션)은 코드베이스 탐색만으로 파악 불가능. 잘 작성된 AGENTS.md는 4% 향상도 실질적으로 의미 있을 수 있음. 팀 단위 프로젝트에서 일관성 유지 목적은 성능 외 다른 가치가 있음.

### 주장 2: "모델은 코드베이스에서 스스로 정보를 잘 찾는다"

- **주장**: 현대 LLM은 package.json, README 등에서 필요한 정보를 자율적으로 탐색하므로 AGENTS.md의 정보 대부분은 중복이다.
- **제시된 근거**: 깨진 UI 수정 데모, init 실행 결과 관찰.
- **근거 유형**: 경험/시연
- **한계**: 소규모 프로젝트 기준. 수십만 줄 이상의 모노레포나 여러 서비스가 얽힌 대형 코드베이스에서는 탐색 비용이 훨씬 크고 오탐 가능성도 높음.
- **반론/대안**: 대형 레포에서 핵심 진입점(key entry points)을 명시하면 탐색 시간을 단축하고 오류를 줄일 수 있다. 탐색 자체도 토큰 비용을 소모하므로 상황에 따라 AGENTS.md로 탐색을 건너뛰는 것이 효율적일 수 있음.

### 주장 3: "AGENTS.md보다 코드베이스 자체를 개선하라"

- **주장**: 유닛 테스트, 타입 체크, 피드백 시스템을 구축하는 것이 AGENTS.md 관리보다 높은 ROI.
- **제시된 근거**: 개인 경험과 직관.
- **근거 유형**: 의견
- **한계**: 코드베이스 구조 개선은 AGENTS.md 작성보다 훨씬 높은 초기 비용이 필요. 레거시 프로젝트나 단기 프로토타입에서는 현실적이지 않을 수 있음.
- **반론/대안**: 두 전략은 상호 배타적이지 않다. AGENTS.md는 코드베이스 개선이 완료될 때까지의 "임시방편"으로 가치가 있다. 화자 본인도 이를 인정함.

### 주장 4: "컨텍스트에 넣지 않는 것이 넣는 것보다 낫다"

- **주장**: 불필요한 정보가 컨텍스트에 있으면 모델의 주의를 분산시킨다.
- **제시된 근거**: "분홍 코끼리" 비유, TanStack 사례.
- **근거 유형**: 경험 + 비유
- **한계**: LLM의 어텐션 메커니즘은 인간의 인지와 다름. "컨텍스트에 있으면 생각한다"는 비유가 트랜스포머 아키텍처에 그대로 적용되는지는 불분명. 관련 없는 정보의 실제 성능 영향은 태스크와 모델 아키텍처에 따라 다를 수 있음.
- **반론/대안**: Long context 모델은 관련 없는 정보를 무시하는 능력이 향상되고 있음. "Lost in the Middle" 문제는 실재하지만, 최신 모델에서는 개선되고 있다는 연구도 있음.

- **주장**: "컨텍스트 파일 제공 시 에이전트 비용이 20% 이상 증가했다" | **검증 필요 이유**: 어떤 태스크셋, 어떤 모델, 어떤 컨텍스트 파일 길이 기준인지 미명시. 저자의 자체 실험에서도 같은 수치가 나왔지만 이는 단일 1회성 실험 | **검증 방법**: 동일 태스크에 AGENTS.md 길이를 달리하며 실제 토큰 소비량 측정 비교. Anthropic/OpenAI API 토큰 usage 로그 분석

- **주장**: "최신 모델은 에이전트 MD가 없어도 코드베이스에서 필요한 정보를 스스로 찾는다" | **검증 필요 이유**: 소규모 프로젝트 기준이며, 대형 모노레포(100만+ 줄)에서의 성능은 별도 검증 필요 | **검증 방법**: SWE-bench 등 대형 레포 벤치마크에서 컨텍스트 파일 유무에 따른 성공률 비교. 탐색 비용(토큰 수) 측정
