## 경쟁 가설 (Competing Hypotheses)

**H1**: 바둑이의 공식 규칙을 그대로 디지털화하면 초보자 진입 장벽이 높아 사용자 이탈이 높을 것이다.
- 검증 근거: 성공한 카드 게임들의 규칙 단순화 패턴

**H2**: 실제 돈을 다루지 않는 점이 한국 사용자의 몰입감을 크게 감소시킬 것이다.
- 검증 근거: 가상 화폐 시스템이 성공한 게임 사례 분석

**H3**: AI 상대 매칭보다는 PvP 소셜 루프가 장기 인게이지먼트의 핵심이다.
- 검증 근거: 카드 게임의 PvP vs AI 비율, 리더보드/토너먼트 효과

**H4**: 안티치트 시스템 없이는 온라인 포커 게임이 불가능하다.
- 검증 근거: 모바일 카드 게임의 부정행위 유형, 기술적 방어 수준

## 영역 1: 바둑이(Badugi) 공식 규칙 정리

### 1.1 게임 기본 구조

**게임 개요**
- 플레이어 수: 2-8명
- 카드 배분: 각 플레이어 4장 (개인 카드, 비공개)
- 목표: **가장 낮은 핸드로 팟을 이김** (역 순위)
- 베팅 라운드: 4회 (초기 배분 + 3회 드로우)
- 드로우: 3회 (각 라운드마다 0~4장 교환 가능)

[신뢰도: High] 출처:
- Washington State Gambling Commission: https://wsgc.wa.gov/sites/default/files/2023-11/Badugi%20Game%20Rules%205-25-23.pdf
- PokerNews Official: https://www.pokernews.com/poker-rules/badugi.htm
- Upswing Poker: https://upswingpoker.com/badugi-rules-hand-rankings/

### 1.2 핸드 순위 체계 (Hand Rankings)

바둑이의 핸드는 **같은 순위(rank)와 같은 무늬(suit)를 피하는 게임**.

```
4-Card Badugi (최강):
  ┌─ A♠-2♥-3♣-4♦ (최고점 Badugi, 가장 낮음)
  ├─ A♠-2♥-3♣-5♦
  ├─ 2♠-3♥-4♣-5♦
  └─ ... (각 카드 조합별로 높은 값들)

3-Card 이하 (같은 순위/무늬 제거):
  ├─ 3-Card: A♠-2♥-3♣ (4번째 동일 무늬/순위 제거)
  ├─ 2-Card: A♠-2♥ (3,4번째 제거)
  └─ 1-Card: A♠ (2,3,4번째 제거)
```

**순위 규칙**:
1. **Tier 비교** (먼저): 4-Card > 3-Card > 2-Card > 1-Card
   - 4-Card Badugi는 항상 3-Card Badugi를 이김
2. **같은 Tier 내 카드값 비교** (그 다음): 낮은 값이 이김
   - 예: A-2-3-4 vs A-2-3-5 → A-2-3-4 승
3. **동일 핸드 분할**: 드물지만, 동일한 핸드는 팟 분할

**기타 규칙**:
- 에이스(A)는 **낮은 순위** (1)로 계산
- 스트레이트/플러시는 **보너스 없음** (순위 영향 없음)
- 각 플레이어는 매 드로우 후 카드 개수를 공개 (투명성)

[신뢰도: High] 출처:
- Cool Old Games: https://www.coololdgames.com/card-games/poker/badugi/
- Counting Outs: https://www.countingouts.com/badugi-rules-and-basic-strategy/
- Game Rules: https://gamerules.com/rules/badugi-card-game/

### 1.3 드로우 및 베팅 라운드

```
라운드 1: 초기 배분 (4장) → 베팅
라운드 2: 1st 드로우 (0~4장 교환) → 베팅
라운드 3: 2nd 드로우 (0~4장 교환) → 베팅
라운드 4: 3rd 드로우 (0~4장 교환) → 베팅 → 쇼다운
```

**베팅 구조** (표준 포커와 동일):
- 빅블라인드(Big Blind) / 스몰블라인드(Small Blind) 설정
- 각 라운드: Check/Bet/Call/Raise/Fold 가능
- 마지막 라운드: 쇼다운 (남은 모든 플레이어가 카드 공개)

**디지털 게임 단순화 여지**:
- 드로우 시 "자동 제안" UI (최적 카드 교환 추천)
- "빠른 게임" 모드: 드로우 타이머 단축 (3-5초)
- 초보자 튜토리얼: 각 라운드의 의미 명확화

[신뢰도: High] 출처:
- PokerTube: https://www.pokertube.com/article/badugi-poker-rules
- Pennsylvania Rules: https://www.pacodeandbulletin.gov/Display/pacode?file=/secure/pacode/data/058/chapter637a/s637a.21.html

### 1.4 변형 규칙 (Variants)

| 변형명 | 핵심 변경 | 참고사항 |
|--------|----------|----------|
| **Badacey** | Badugi + Ace-to-Five Low (2가지 핸드 동시 평가) | 복잡함, 고급 플레이어용 |
| **Baduci** | Badugi + Deuce-to-Seven Low | 드문 변형 |
| **Triple Draw** | 드로우 5회 (표준 3회 → 5회) | 더 긴 게임 |
| **Limit Badugi** | 베팅 상한선 고정 (포팟 리밋 제거) | 보수적 게임 |

**디지털 게임 설계 권고**:
- **첫 번째 출시**: 표준 Badugi만 (변형 제외)
- **시즌 2+**: Badacey를 2차 게임 모드로 추가 고려
- 변형이 사용자 혼란 증가 → 초기 교육 부담 상승

[신뢰도: Medium] 출처:
- PokerNews variants section: https://www.pokernews.com/poker-rules/badugi.htm

## 영역 2: 디지털 카드 게임 코어 루프 설계

### 2.1 성공한 게임의 코어 루프 분석

**카드 게임의 일반 코어 루프**:

```
┌─────────────────────────────────────────┐
│  매칭 & 핸드 배분 (Deal)                │
└───────────┬─────────────────────────────┘
            ↓
┌─────────────────────────────────────────┐
│  카드 액션 (Draw/Bet/Play)              │
│  - 결정 포인트 (의도적 불확실성)       │
│  - 리스크-리워드 밸런스               │
└───────────┬─────────────────────────────┘
            ↓
┌─────────────────────────────────────────┐
│  결과 평가 & 피드백                     │
│  - 핸드 공개 (쇼다운)                   │
│  - 팟 분배                              │
│  - 보상 (칩/포인트) 즉시 표시           │
└───────────┬─────────────────────────────┘
            ↓
┌─────────────────────────────────────────┐
│  성장 & 진행                            │
│  - 랭킹 상승                            │
│  - 배틀패스 프로그레스                  │
│  - 다음 게임으로 복귀                   │
└─────────────────────────────────────────┘
```

### 2.2 게임별 코어 루프 비교

| 게임 | 세션 길이 | 핵심 재미 | 피드백 속도 | 중독성 |
|------|:--------:|---------|-----------|--------|
| **Hearthstone** | 10-15분 | 덱 시너지 + 상대 압박 | 즉시 (턴 기반) | 높음 |
| **Slay the Spire** | 30-60분 | 싱글 플레이 + 빌드 깊이 | 느림 (깊은 전략) | 중간-높음 |
| **Balatro** | 20-40분 | 포커 핸드 + 조합 폭발 | 빠름 (즉시 배수 표시) | 극도로 높음 |
| **Marvel SNAP** | 3-5분 | 빠른 결정 + 소셜 | 극히 빠름 | 중간 |

**Balatro의 중독성 메커니즘**:
- **짧은 세션**: 20-40분 루프 → 즉시 다음 게임 시작 가능
- **즉각적 피드백**: 각 카드 선택 시 점수 배수가 **실시간 증가** (시각적 쾌감)
- **시너지 폭발**: 예상 밖의 조합이 갑자기 터짐 → 놀라움 + 성취감
- **무한 변수**: 매 게임마다 다른 조합 → 반복 피로 없음

[신뢰도: Medium-High] 출처:
- Digital Edge: https://digitaledge.org/how-balatro-turned-poker-hands-into-the-ultimate-roguelike-addiction
- Gamesub: https://www.gameshub.com/news/features/balatro-roguelike-deckbuilder-2637397/
- Medium: https://batiste.medium.com/designing-a-card-game-5f610a1fcc71

```
┌────────────────────────────────────┐

│     - 쇼다운 시 애니메이션         │

│     - 승/패 명확한 표시           │
│     - 다음 핸드 준비 버튼         │
└──────────┬───────────────────────┘
           ↓
┌────────────────────────────────────┐
│  4. 진행 & 보상                    │
│     - 게임 1판 끝 → ELO ±10-50   │
│     - 일일 미션 진행 확인         │
│     - 배틀패스 XP 적립           │
│     - 즉시 다음 게임 매칭        │
└────────────────────────────────────┘
```

**핵심 설계 원칙**:
1. **세션 길이 5-8분**: 한국 모바일 게이머의 표준 (지하철, 쉬는 시간)
2. **의사결정 포인트 명확화**: 각 드로우마다 "이 핸드로 가는가" 선택
3. **즉각적 보상**: 팟 획득 시 칩 증가를 **크고 명확하게** 표시
4. **다음 게임 진입 장벽 최소화**: "다시 게임" 버튼 1클릭

## 영역 3: 메타게임 시스템 설계 (장기 인게이지먼트)

### 3.1 랭킹 & 매칭 시스템

**ELO vs Glicko-2: 선택 기준**

| 요소 | ELO | Glicko-2 |
|------|-----|---------|
| 복잡도 | 낮음 (기본 계산) | 높음 (volatility rating) |
| 입자 활동 반영 | 약함 | 강함 (RD: Rating Deviation) |
| 시간 팩터 | 없음 | 있음 (비활동 플레이어 자동 상향) |
| 온라인 포커 사용 | 많음 | 소수 (고급 플랫폼) |
| 단계별 도입 | 초기 출시 | 시즌 2+ |

**권장 구현: Hybrid Approach**
- **출시 초기**: ELO 기반 (단순성, 빠른 계산)
- **시즌 1 후기**: Glicko-2로 마이그레이션 (활동성 반영)

**ELO 구현 상세** (GGPoker 참고):

```
플레이어 A (Rating: 1800) vs 플레이어 B (Rating: 1500)
손 승리 시 포인트:
  - A가 B를 이기면: +20점 (낮은 상대)
  - B가 A를 이기면: +50점 (높은 상대)

매 시즌 마다 ELO 초기화 (Seasonal Reset):
  - 상위 100명: 점수 유지 (정상)
  - 나머지: 50% 리셋 (50% 유지)
  - 예: 2000 Rating → 1500으로 시즌 시작
```

**레벨 티어** (GGPoker Spin & Gold 참고):

```
Tier 구조:
  ├─ E (입문):  800 ~ 1,200
  ├─ D (초급):  1,200 ~ 1,500
  ├─ C (중급):  1,500 ~ 3,000
  ├─ B (고급):  3,000 ~ 4,000
  ├─ A (엘리트): 4,000 ~ 5,000
  ├─ Master:    5,000+
  └─ Grand Master: TOP 100 플레이어
```

- 보상 = 경쟁 우위가 아닌 **미학적 차별** (스킨 중심)

[신뢰도: Medium] 출처: 성공 게임 모범 사례 (Hearthstone Battle Pass 분석)

## 영역 4: Baduki 디지털화 시 핵심 설계 결정

### 4.1 가상 화폐 시스템 (실제 돈 vs 가상 화폐)

**H2 검증**: 실제 돈을 다루지 않으면 몰입감 감소?

**리서치 결과**:

```
모바일 게임 수익 구조 (2024):
  ├─ 무료 다운로드 비율: 97%
  ├─ 구매 전환율: 3-5% (매우 낮음)
  └─ 평균 결제 금액: $0 ~ $100 (극도로 분포)

한국 특수성:
  ├─ 사행성 규제 강화 (2023-2024)
  │  └─ 실제 돈 도박 게임 = 불법 가능성
  ├─ 포커 게임의 합법성 논쟁
  │  └─ 바둑이 = 순수 게임 vs 도박 논쟁 중
  └─ 해결책: 100% 가상 화폐 (실제 환전 불가)
```

- **사행성 회피**: 실제 환전 불가 명시

- **법적 안전성**: "게임 내 가상 화폐" 명확화
