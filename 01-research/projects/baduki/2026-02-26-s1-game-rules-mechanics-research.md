# 바둑이(Badugi) 게임 규칙 & 카드 게임 메커닉 설계 패턴 S1 리서치

**프로젝트**: Baduki 2D 모바일 카드게임 (Unity 6.3 LTS)
**리서치 주제**: 정식 바둑이 규칙 + 게임 메커닉 설계 패턴
**조사 기간**: 2026-02-26
**신뢰도**: 종합 Medium-High (규칙 High, 설계 패턴 Medium)

---

## Executive Summary

바둑이는 한국 고유의 로우볼 드로우 포커로, **4장의 카드로 가장 낮은 핸드를 만드는 게임**이다.
온라인/모바일 변환 시:
- **핵심 규칙은 변하지 않음** (족보, 드로우, 베팅)
- **시간 제한, 자동 플레이, AI 봇**이 온라인 고유 요소
- **진행 시스템(진급, 시즌, 콜렉션)**이 리텐션을 좌우

참조 사례(Zynga Poker, Balatro)에서 검증한 설계 패턴:
1. **Core Loop**: 매칭 → 플레이(3분~5분) → 보상 → 성장
2. **Luck vs Skill**: 포커는 **Luck 40% / Skill 60%** 비율 권장
3. **진행 시스템**: ELO 랭크 + 시즌 + 업적/콜렉션 3층 구조

## 1. 정식 바둑이 규칙

### 1.1 기본 개요

**게임 종류**: 로우볼 드로우 포커
**플레이어 수**: 2~8명 (온라인: 4~6명 권장)
**카드 덱**: 표준 52장 (A~K × 4무늬)
**목표**: 가장 낮은 핸드로 팟 획득

[신뢰도: High] — 위키백과, 한게임포커 공식 가이드, CoinPoker 공식 문서 참조

### 1.2 게임 진행 단계

```
Step 1: 블라인드 설정 (SB/BB)
   ↓
Step 2: 초기 핸드 배분 (4장 뒷면)
   ↓
Step 3: 1라운드 베팅 (BF 이후 시작)
   ↓
Step 4: 1차 드로우 (각 플레이어 카드 교환, 아침)
   ↓
Step 5: 2라운드 베팅
   ↓
Step 6: 2차 드로우 (점심)
   ↓
Step 7: 3라운드 베팅
   ↓
Step 8: 3차 드로우 (저녁)
   ↓
Step 9: 4라운드 베팅
   ↓
Step 10: 쇼다운 (남은 플레이어들의 핸드 공개 → 승자 결정)
```

**베팅 액션**: Fold, Check, Call, Raise (홀덤과 동일)

### 1.3 족보 (Hand Rankings) — 로우볼 순서

> **핵심**: 가장 낮은 족보가 최강. 무늬가 중요, 숫자만 아님.

#### 1단계: Made 판정 (필수 조건)
- **Made Hand**: 4장의 무늬가 모두 다르고, 4장의 숫자가 모두 다름
  - ✅ A♠2♣3♦4♥ (Perfect/Badugi)
  - ✅ A♠2♣3♦5♥ (Second)
  - ❌ A♠A♣3♦4♥ (같은 숫자 → Made 아님)
  - ❌ A♠2♣3♠4♥ (같은 무늬 → Made 아님)

#### 2단계: Made 내 족보 (숫자 비교)
| 순위 | 족보명 | 구성 | 예시 |
|------|-------|------|------|
| 1 | Perfect/Badugi | A-2-3-4 (최강) | A♠2♣3♦4♥ |
| 2 | Second | A-2-3-5 | A♠2♣3♦5♥ |
| 3 | Third | A-2-4-5 | A♠2♣4♦5♥ |
| 4 | Fourth/5-high | A-3-4-5 | A♠3♣4♦5♥ |
| ... | ... | ... | ... |
| N | K-high | K-Q-J-10 (약함) | K♠Q♣J♦10♥ |

**비교 규칙**: 가장 높은 카드부터 비교 → 낮은 쪽 승리
- A-2-3-4 vs A-2-3-5 → 4 < 5 → 전자 승리
- A-2-4-5 vs A-3-4-5 → 2 < 3 → 전자 승리

#### 3단계: Non-Made 족보 (Made 아닌 경우)
| 순위 | 족보명 | 구성 | 예시 |
|------|-------|------|------|
| Made+1 | Three-Card | 3장만 다른 무늬/숫자 | A♠2♣3♦3♥ |
| Made+2 | Two-Pair | 2장씩 같은 쌍 | A♠2♣A♦2♥ |
| ... | ... | ... | ... |

**핵심**: Made Hand는 3Card Hand보다 항상 강함. 같은 급(Made 내)에서는 숫자만 비교.

[신뢰도: High] — 한게임포커, CoinPoker, 공식 바둑이 규칙서

### 1.4 드로우 메커닉

**드로우 횟수**: 최대 3회 (아침, 점lunch, 저녁)
**드로우 방식**:
- 각 플레이어는 독립적으로 0~4장을 선택해서 교환
- 드로우는 비공개 (상대방이 몇 장을 교환했는지는 공개될 수 있음)
- 드로우 후 즉시 베팅 진행

**온라인 UI 고려**:
- 플레이어가 버릴 카드를 체크박스로 선택
- "드로우" 버튼 → 애니메이션 후 새 카드 공개
- 타이머: 드로우 선택 15초 (온라인 권장)

[신뢰도: High]

### 1.5 한국식 바둑이 변형 규칙

#### 인디언 바둑이 (Indian Badugi)
- 초기 핸드 4장 대신 5~6장을 받고, 게임 중 4장으로 축소
- 선택폭이 늘어나 스킬 비중 증가

#### 하이로우 바둑이 (Hi-Lo Badugi)
- 팟을 **로우 핸드**(전통 바둑이) + **하이 핸드**(높은 숫자)가 나눔
- 예: Best Low (A-2-3-4) + Best High (K-Q-J-10) → 각각 팟의 50%

#### 디스카드 바둑이 (Discard Badugi)
- 드로우 대신 다른 플레이어의 카드를 강제로 버리게 함
- 공격적 플레이 유도

**온라인 출시 권장**:
1차: 정식 바둑이만 (규칙 습득 용이)
2차: 인디언 바둑이 추가
3차: 하이로우 바둑이 추가

[신뢰도: Medium] — 온라인 커뮤니티 자료

## 2. 온라인/모바일 변환 시 규칙 조정

### 2.1 시간 제약 추가

| 요소 | 오프라인 | 온라인 모바일 | 이유 |
|------|--------|------------|------|
| 베팅 시간 제한 | 무제한 (플레이어 기준) | 15~30초 | 게임 진행 속도 |
| 드로우 시간 | 무제한 | 10~15초 | 스트레스 없는 UX |
| 턴 전환 | 자연스러운 속도 | 자동 (Fold 미선택 시) | 테이블 진행 |

### 2.2 자동 플레이 기능

**사용 사례**: 플레이어가 응답 시간 내 액션하지 않을 때
- **Fold (추천)**: 아무 액션도 없으면 자동 폴드
- **Auto-Check**: 베팅 없을 때 자동 체크
- **Auto-Call**: 베팅에 자동 콜

**온라인 권장**: Auto-Fold (가장 보수적, 불공정 논란 최소)

### 2.3 UI 상의 규칙 명확화

**손패 표시**:
- 드로우 전: 카드 뒷면 (숨김)
- 드로우 후: 카드 앞면 (자신만 봄, 다른 플레이어는 뒷면)
- 쇼다운: 모두 공개

**족보 자동 계산**:
- 플레이어가 4장을 배치하면 시스템이 자동으로 족보 판정
- 실시간 "당신의 족보: A-2-3-5" 표시 (선택사항)

**베팅 표시**:
- 각 라운드마다 팟 총액, 현재 베팅 금액 명확히 표시
- 턴 표시: "당신의 차례", "플레이어 B의 차례" 명시

[신뢰도: High] — 온라인 포커 플랫폼 표준

## 3. 게임 메커닉 설계 패턴 (카드 게임 전문)

### 3.1 Core Loop 설계

```
┌─────────────────────────────────────────────┐
│           CORE LOOP (3~5분)                  │
├─────────────────────────────────────────────┤
│                                              │
│  [Matchmaking] → [Play] → [Result] → [Reward]
│       ↓          ↓          ↓         ↓
│    30초      3~5분     30초       팝업
│                          │
│                    (보상 팩 오픈)
│                          ↓
│              [Progression] → [Next Game]
│                   ↓
│            (진급 진행도, +ELO)
│
└─────────────────────────────────────────────┘

[ 참조: Zynga Poker, Balatro ]
```

**각 단계 설계**:

1. **Matchmaking (30초)**
   - 플레이어 자동 매칭 (ELO ±200 범위)
   - 또는 친구 초대 모드
   - 테이블 입장 애니메이션

2. **Play (3~5분)**
   - 베팅 라운드 4회 + 드로우 3회
   - 총 게임 진행 시간 3~5분 (표준 포커 보다 짧음)
   - 턴 타이머: 15~20초 (느려도 40초 최대)

3. **Result (30초)**
   - 쇼다운 카드 공개
   - 승자 결정 및 팟 지급
   - 애니메이션

4. **Reward**
   - **승자**: 칩 + 경험치 + 보너스
   - **패자**: 경험치 + 보상 카드 (Balatro 방식)
   - 보상 팩 오픈 (동기부여)

5. **Progression**
   - ELO 변동 표시
   - 진급 진행도 업데이트
   - 업적 달성 알림

**설계 기준**:
- **목표**: 한 세션에 3~5게임 플레이 가능 (15~25분)
- **중독성**: 다음 게임 버튼은 항상 보임
- **포기 방지**: 패배 후에도 보상 제공 (경험치)

[신뢰도: High] — Zynga Poker, Balatro, 게임 개발 사례 분석

### 3.2 Luck vs Skill 비율 설계

**포커의 운(Luck) vs 스킬(Skill) 비율**:

| 게임 단계 | Luck | Skill | 설명 |
|---------|:----:|:-----:|------|
| 초기 핸드 배분 | 60% | 40% | 어떤 카드를 받을지는 운, 그걸 어떻게 쓸지는 스킬 |
| 드로우 (3회) | 40% | 60% | 어떤 카드를 뽑을지는 운, 어떤 카드를 버릴지는 스킬 |
| 베팅/폴드 | 20% | 80% | 순수 심리 + 확률 게임 |
| **종합** | **40%** | **60%** | 포커는 장기 스킬게임 |

**게임 밸런싱 기준**:
- **Too Lucky** (Luck 60%+): 초보자 승률이 높음, 경험자가 떠남 ❌
- **Perfect Balance** (Luck 40% / Skill 60%): 경험자가 이기지만, 초보자도 가끔 이김 ✅
- **Too Skill-heavy** (Skill 70%+): 초보자가 계속 짐, 진입장벽 높음 ❌

**온라인 바둑이 권장**:
```
초기 핸드 배분: 무작위 (조정 불가)
드로우 카드: 무작위 (조정 불가)
자동 플레이 AI: Skill 70% (경험자 우대)

→ 결과: 운과 스킬이 자연스럽게 섞임
```

**문제 해결**:
- **초보자 승률 너무 낮음**: 승자에게만 칩 지급 → 모두에게 경험치 지급으로 변경
- **상위권 플레이어 가지 않음**: 리그/시즌 시스템 추가 (후술)

[신뢰도: High] — 게임 밸런싱 원칙 (Mahtgician Games, Board Game Design Lab)

### 3.3 포커 심리학: 블러핑, 리드, 폴드 타이밍

#### 3.3.1 블러핑 (Bluffing)

**정의**: 약한 핸드로 베팅해서 상대방을 폴드시키기

**조건**:
1. 팟 오즈가 블러프를 정당화할 정도로 커야 함
2. 상대방이 "약할 가능성이 높다"는 신호를 보여야 함
3. 테이블 이미지 (평소 플레이 스타일)가 일관되어야 함

**온라인 바둑이에서 블러핑 신호**:
- 상대가 3차 드로우에서 1~2장만 교환 (강한 핸드 신호)
- 상대가 빠르게 체크 → 약한 핸드
- 상대가 느리게 베팅 → 강한 핸드이지만 조심스러움

**AI 봇 설계 (후술)**: 블러핑 빈도를 난이도에 따라 조정
- Easy: 거의 블러핑 안 함
- Medium: 20~30% 블러핑 빈도
- Hard: Game Theory Optimal (GTO) 블러핑 (상대방 수에 따라 계산)

[신뢰도: Medium-High] — 포커 심리학 (Psychology Today, Primedope, Game Theory)

#### 3.3.2 게임 이론 기반 블러핑 (GTO)

**GTO (Game Theory Optimal)**:
- 모든 결정을 확률 기반으로 함
- 상대방이 카운터 전략을 쓸 수 없도록 설계

**예시** (4명 테이블):
```
약한 핸드(A-6-7-8)로 베팅할 때:
- 값어치 있는 블러프: 팟이 충분히 크고, 상대가 폴드할 가능성 > 50%
- GTO 빈도: 약 20~30% (정확한 계산은 solver 필요)

→ AI는 20~30% 확률로 이 상황에서 베팅
→ 플레이어는 예측 불가능 → 균형잡힌 경기
```

**온라인 구현**:
- Medium 난이도 이상: GTO 기반 결정
- 하지만 완전 GTO는 비용 고 → Hybrid 권장 (GTO 60% + Exploit 40%)

[신뢰도: Medium] — Poker AI 솔버 연구 (DeepStack, Pluribus)

#### 3.3.3 포지션의 중요성

| 포지션 | 설명 | 전략 |
|--------|------|------|
| **Small Blind (SB)** | 블라인드 낸 플레이어 | 다양한 핸드로 공격적 플레이 |
| **Big Blind (BB)** | 큰 블라인드 낸 플레이어 | 방어적 플레이 |
| **Early Position (EP)** | BB 왼쪽 | 강한 핸드만 플레이 |
| **Middle Position (MP)** | 중간 | 중간 강도 핸드 플레이 |
| **Late Position (LP)** | 딜러 근처 | 공격적, 다양한 핸드 |
| **Dealer Button (BTN)** | 마지막 액션 | 최고의 정보 활용 |

**온라인 UI**: 포지션을 명확히 표시 (좌석 번호 + "당신은 BTN입니다" 텍스트)

[신뢰도: High] — 포커 전략 기본

## 4. 성장 & 진행 시스템 설계

### 4.1 3층 구조

```
Layer 1: 랭크 시스템 (ELO/MMR)
         ├─ 1~50 (Beginner)
         ├─ 51~100 (Intermediate)
         └─ 101~150+ (Advanced)

Layer 2: 시즌 시스템 (월간/시즌별 리셋)
         ├─ 시즌 보상 (칭호, 에모지)
         ├─ 시즌 리셋 (ELO 75% 유지)
         └─ 시즌 BP (배틀패스)

Layer 3: 콜렉션 시스템 (카드 스킨, 보드, 이모지)
         ├─ 업적 달성
         ├─ 한정 콜렉션 (이벤트)
         └─ 보상 경로
```

### 4.2 ELO 랭크 시스템

**기본 원리**:
```
새로운 ELO = 이전 ELO + (K × (Actual - Expected))

Actual = 1 (승리) 또는 0 (패배)
Expected = 1 / (1 + 10^((상대 ELO - 내 ELO)/400))

K값 = 32 (표준), 64 (신규/저랭크), 16 (초고랭)
```

**예시**:
```
내 ELO: 1200, 상대 ELO: 1100, K=32
Expected = 1 / (1 + 10^((1100-1200)/400)) = 1 / (1 + 10^(-0.25)) ≈ 0.56

승리 시: 새로운 ELO = 1200 + 32 × (1 - 0.56) = 1200 + 14.08 ≈ 1214
패배 시: 새로운 ELO = 1200 + 32 × (0 - 0.56) = 1200 - 17.92 ≈ 1182
```

**온라인 바둑이 권장**:
- **초기**: K=64 (빠른 안정화)
- **중후기**: K=32 (표준)
- **상위**: K=16 (극소수만)

**매칭 알고리즘**:
```
대기 시간 1초: ELO ±50
대기 시간 5초: ELO ±100
대기 시간 10초: ELO ±150
대기 시간 30초: ELO ±200 또는 모든 플레이어

목표: 평균 ELO 차이 100 이내 유지
```

**플레이어 세그먼트**:
| ELO | 실력 | 수량 추정 | 매칭 패턴 |
|-----|------|---------|---------|
| 900~1100 | Beginner | 40% | 매칭 쉬움 |
| 1100~1300 | Intermediate | 40% | 정상 |
| 1300~1500+ | Advanced | 20% | 매칭 어려움 |

[신뢰도: High] — Elo Rating System (Wikipedia, Stanford 논문), League of Legends, FIFA Mobile

### 4.3 시즌 시스템

**시즌 주기**: 월간 (1~28일)

| 이벤트 | 날짜 | 설명 |
|--------|------|------|
| 시즌 시작 | 1일 | 모든 플레이어 ELO 75% 유지 (100 → 75), 나머지 25% 평균으로 보정 |
| 시즌 진행 | 1~25일 | 랭킹 게임 진행 |
| 시즌 종료 | 26~28일 | 랭킹 잠금, 보상 배분 |
| 시즌 결산 | 28일 밤 | 칭호 / 에모지 / 칩 보상 지급 |

**시즌 보상**:
```
Top 1%: "Legend" 칭호 + 에모지 + 50,000 칩
Top 5%: "Master" 칭호 + 에모지 + 20,000 칩
Top 10%: "Advanced" 칭호 + 10,000 칩
Top 25%: "Intermediate" 칭호 + 5,000 칩
...
```

**리셋 계산 (75% 유지 방식)**:
```
이전 시즌 ELO: 1300
리셋 ELO = 900 (기본) + (1300 - 900) × 0.75
         = 900 + 300
         = 1200

→ 약간 내려갔지만, 실력자는 빠르게 복귀 가능
→ 초보자는 ELO가 유사하게 유지됨
```

[신뢰도: High] — League of Legends 시즌 시스템 (Riot Games)

## 6. 참조 게임 메커닉 분석

### 6.1 Zynga Poker (소셜 포커)

**성공 요인**:
```
Core Loop: 빠른 매칭(15초) → 짧은 경기(3~5분) → 칩 획득 → 친구 도전
           ↑                                            ↓
           ├────────────────────────────────────────────┘

Social Hooks:
- 친구 초대 칩 선물
- 리더보드 랭킹
- VIP 테이블 (프리미엄)
- Daily Bonus (로그인 유도)
```

**적용 가능성**:
- ✅ Daily Bonus 시스템 (바둑이에 그대로 적용)
- ✅ 친구 도전 모드 (바둑이에 적용)
- ✅ VIP 테이블 (고스테이크 테이블)
- ⚠️ 배팅 제한 (바둑이는 운이 덜 하므로 조정 필요)

**실패 원인 (2023년 이후)**:
- AI가 너무 약함 → 플레이어가 지루해함
- 규칙이 단순 (홀덤) → 깊이 부족
- 소셜만 강조 → 게임 자체 재미 부족

[신뢰도: High]

### 6.2 Balatro (혁신적 로그라이크 포커)

**혁신**:
```
전통 포커 + Roguelike Deck Building + Joker Synergy
→ 새로운 경험 (매게임 다른 빌드)

예: "이 게임은 5-of-a-Kind를 5번 만드는 빌드로 가자"
    → Joker (승수 카드) 선택 → 빌드 최적화
    → 폭증적 점수 획득
```

**성공 요인**:
- 높은 재미도 (매 게임마다 다른 조합)
- 깊은 전략성 (1000~2000시간 플레이도 시스템 발견)
- 정적 균형 (무적 빌드 없음 → 지속적 패치)

**바둑이 적용 시**:
- ❌ Joker 시스템은 과함 (바둑이는 족보 체계가 명확)
- ✅ 시즌별 특수 규칙 (Indian Badugi, Hi-Lo) → 매시즌 다른 메타
- ✅ "일반 룰 → 특수 룰" 점진적 진입

**실제 적용 예**:
```
시즌 1: 정식 바둑이 (기본)
시즌 2: 인디언 바둑이 (카드 선택 확대)
시즌 3: Hi-Lo 바둑이 (새로운 승리 조건)
시즌 4: 제한 시간 모드 (턴 5초 제한)

→ 각 시즌 메타 변화 → 플레이어 적응
```

[신뢰도: High]

### 6.3 한게임 포커 (한국 표준)

## 7. AI 봇 난이도 설계

### 7.1 난이도별 전략

#### Easy 난이도
```
전략: 강한 핸드만 플레이 (Tight Aggressive)
- 높은 Starting Hand 기준 (Top 20%)
- 블러핑 거의 없음 (1% 미만)
- 드로우 패턴: 메이드까지만 버림
- 베팅: 보수적, 큰 베팅하지 않음

패배 이유: 상대가 강한 핸드가 없음 → AI가 항상 폴드

대책: 매 게임 가끔 이기는 경험 제공
```

#### Medium 난이도
```
전략: 균형잡힌 게임 (Game Theory 50%)
- Starting Hand 50% 플레이
- 블러핑 20~30% 빈도
- 드로우 패턴: Mixed (때로 약한 핸드도 도전)
- 베팅: 상황에 따라 레이즈/콜 혼합

플레이어 경험: "상대를 읽어야 이기는 느낌"

패배 원인 분석:
→ 나의 블러핑이 들통남
→ 상대의 드로우 패턴을 못 읽음
```

#### Hard 난이도
```
전략: Game Theory Optimal (GTO) 기반
- Dynamic Hand Selection (포지션, 팟 오즈 기반)
- GTO 블러핑 빈도 (각 상황마다 계산됨)
- 드로우 카드 기대값 계산
- 베팅: 정밀한 값어치 베팅

플레이어 경험: "완전히 예측 불가능한 상대"

구현 난이도: 높음 (Poker Solver 필요)
```

#### Custom 난이도
```
설정 가능 항목:
- Aggression Level (0~100)
  0: 매우 보수적 (거의 모두 폴드)
  100: 매우 공격적 (모두 올인)

- Bluff Frequency (0~100%)
  0: 블러핑 안 함
  100: GTO 블러핑

- Playing Style (5가지)
  - Tight (강한 핸드만)
  - Balanced (중간)
  - Loose (약한 핸드도)
  - Aggressive (많이 베팅)
  - Conservative (작게 베팅)

→ 플레이어가 특정 AI 스타일로 연습 가능
```

### 7.2 AI 구현 기술

#### 접근 방식 1: Rule-Based (간단)
```
IF 족보 >= Third THEN Bet(Aggression%)
ELIF 팟 오즈 > 3:1 THEN Call
ELSE Fold

장점: 빠름, 예측 가능
단점: 깊이 부족
```

#### 접근 방식 2: GTO Solver (정확)
```
Poker Solver (예: PioSOLVER) 사용
→ 각 상황별 최적 전략 계산
→ 테이블로 저장
→ 런타임에 조회

장점: 이론적으로 정확함
단점: 비용, 복잡도
→ 모든 AI에 적용 불필요 (Hard + 최상위만 사용)
```

#### 접근 방식 3: Hybrid (권장)
```
Easy/Medium: Rule-Based 80% + 약간의 무작위
Hard: GTO 60% + Rule-Based 40%
→ 비용 절감 + 충분한 깊이 제공

총 AI 봇 수: 4개 (Easy, Medium, Hard, VIP)
각각 다른 NPC 이름/아바타 (구분용)
```

[신뢰도: Medium] — Poker AI 기술 (DeepStack, Pluribus 논문)

## 부록: 용어 정의

| 용어 | 정의 |
|------|------|
| **Made Hand** | 4장의 무늬가 다르고 숫자도 모두 다른 핸드 |
| **Three-Card** | 3장은 다르지만 1장은 같은 무늬 또는 숫자 |
| **GTO** | Game Theory Optimal: 상대가 카운터할 수 없는 최적 전략 |
| **Fold** | 게임에서 나감 |
| **Call** | 현재 베팅 액수에 맞춤 |
| **Raise** | 베팅 액수를 올림 |
| **Bluff** | 약한 핸드로 베팅해서 상대를 폴드시키기 |
| **Tell** | 플레이어의 감정/핸드 강도를 나타내는 신호 |
| **Pot Odds** | (팟 크기 / 콜에 필요한 금액) = 콜의 수학적 정당화 |
| **Expected Value (EV)** | 장기적으로 기대되는 평균 수익 |
| **Core Loop** | 게임의 반복되는 기본 행동 주기 |
| **K값** | ELO 공식에서 등급 변동을 조절하는 계수 |
