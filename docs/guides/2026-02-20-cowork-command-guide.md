# Cowork 슬래시 커맨드 가이드

**작성일**: 2026-02-20
**목적**: Claude Cowork에서 슬래시 커맨드로 업무를 빠르게 처리하는 실용 가이드
**대상**: Claude Desktop(Cowork) 사용자 — 개발 지식 불필요
**커맨드 수**: 17개 (Cowork 사용 가능) + 1개 (CLI 전용)

---

## 슬래시 커맨드란?

슬래시 커맨드는 복잡한 요청을 한 줄로 자동화한 것입니다. 매번 긴 지시를 쓰는 대신, `/커맨드명`과 간단한 설명만 입력하면 전문가 수준의 결과물을 자동으로 만들어 줍니다.

예를 들어 "AI SaaS 시장에 대해 조사해줘. 시장 규모, 경쟁사, 트렌드를 포함해서..."라고 길게 쓰는 대신:

```
/research AI SaaS 시장
```

이 한 줄이면 시장 규모, 경쟁사, 트렌드, 기회/위험 요인까지 체계적으로 정리된 리포트가 자동으로 생성됩니다.

## 업무별 커맨드 상세

### 1. 시장조사

#### `/research` — 시장조사 리포트

| 항목 | 내용 |
|------|------|
| **언제 사용** | 새 사업 아이디어가 떠올랐을 때, 특정 시장의 규모와 트렌드가 궁금할 때 |
| **입력** | 조사하고 싶은 주제 (시장명, 기술명, 산업명 등) |
| **결과물** | 시장 규모, 주요 경쟁사, 트렌드, 기회/위험 분석 리포트 |
| **저장 위치** | 시장조사 폴더 (`01-research/market-data/`) |

**요청 예시**:
- `/research AI 자동 번역 시장` — 글로벌 AI 번역 시장 규모, 성장률, 주요 서비스 분석
- `/research 한국 프리랜서 플랫폼 시장` — 국내 프리랜서 플랫폼 현황과 기회 분석

#### `/competitor` — 경쟁사 심층 분석

| 항목 | 내용 |
|------|------|

| **저장 위치** | 경쟁사 분석 폴더 (`01-research/competitors/`) |

**요청 예시**:
- `/competitor Linear` — 프로젝트 관리 도구 Linear의 기능, 가격, 고객 반응 분석

**요청 예시**:

| 항목 | 내용 |
|------|------|
| **언제 사용** | 쌓인 고객 피드백(리뷰, 설문, CS 티켓 등)에서 패턴과 인사이트를 뽑고 싶을 때 |
| **입력** | 피드백 소스 설명 또는 피드백 내용 직접 입력 |
| **결과물** | 카테고리 분류, 감정 분석, 핵심 패턴 Top 5, 우선순위 백로그 |
| **저장 위치** | 고객지원 폴더 (`10-operations/support/`) |

**요청 예시**:
- `/feedback-analysis App Store 리뷰 최근 20건` — 리뷰 패턴 분석 + 개선 우선순위
- `/feedback-analysis 이번 달 CS 티켓 요약: 로그인 오류 5건, 결제 문의 3건, 기능 요청 7건` — 직접 입력도 가능

---

#### `/incident` — 장애 회고

| 항목 | 내용 |
|------|------|
| **언제 사용** | 서비스 장애가 발생한 후 원인 분석과 재발 방지책을 정리할 때 |
| **입력** | 장애 요약 (예: 결제 오류, API 다운) |
| **결과물** | 타임라인, 5 Whys 근본 원인 분석, 단기/중기/장기 재발 방지책 |
| **저장 위치** | 장애 기록 폴더 (`10-operations/incidents/`) |

**요청 예시**:
- `/incident 결제 API 30분 다운` — 장애 상세 정보를 질문한 후 회고 문서 자동 생성
- `/incident 회원가입 페이지 SSL 인증서 만료` — 타임라인 + 근본 원인 + 재발 방지책

## 업무 시나리오 워크플로우

커맨드를 조합하면 복잡한 업무도 단계별로 처리할 수 있습니다.

### 시나리오 1: 새 제품 아이디어 검증 (1~2일)

```
/research AI 코드리뷰 시장              ← 시장이 충분히 큰지 확인
      ↓
/competitor CodeRabbit                  ← 핵심 경쟁사 분석
/competitor Codacy                      ← 또 다른 경쟁사 분석
      ↓
/lean-canvas AI 코드리뷰 SaaS           ← 비즈니스 모델 1장 정리
      ↓
/prd AI 코드리뷰 자동화 도구             ← 구체적 기획서 완성
```

**포인트**: 앞 단계의 결과가 자동으로 참조되므로, 갈수록 더 구체적인 결과물이 나옵니다.

---

### 시나리오 2: 신규 서비스 런칭 마케팅 (3~5일)

```
/gtm 코드리뷰 SaaS, B2B 개발팀          ← 시장 진입 전략 수립
      ↓
/pricing 코드리뷰 SaaS, B2B             ← 가격 모델 결정
      ↓
/seo-audit AI 코드리뷰 도구              ← 검색 노출 전략 수립
      ↓
/campaign 서비스 런칭, 스타트업 CTO       ← 마케팅 캠페인 설계
      ↓
/email-sequence 무료체험→유료전환         ← 전환 유도 이메일 작성
      ↓
/lead-research 코드리뷰 SaaS, 50인+ 개발팀  ← 잠재 고객 탐색
```

---

### 시나리오 3: 월간 운영 루틴 (매월 반복)

| **장애 발생 시** | `/incident` | 장애 회고 작성 |

## 알아두면 좋은 것

### 입력을 자세히 쓸수록 결과가 좋습니다

| 간단한 입력 | 구체적인 입력 |
|------------|-------------|
| `/research AI 시장` | `/research 한국 중소기업 대상 AI 문서 자동화 시장, B2B 중심` |
| `/competitor Notion` | `/competitor Notion — 프로젝트 관리 기능 중심으로, 10인 이하 팀 관점` |
| `/campaign 런칭` | `/campaign 3월 정식 런칭 이벤트, 타겟은 50인 이하 스타트업 CTO` |

구체적으로 쓸수록 시장 범위, 분석 관점, 타겟 고객이 명확해져서 실제 활용 가능한 결과가 나옵니다.

### 이전 결과를 자동으로 참조합니다

커맨드를 연달아 실행하면, 앞서 저장된 파일을 자동으로 찾아 참고합니다.

- `/research` → `/competitor` 실행 시: 시장조사 결과를 바탕으로 더 깊은 분석
- `/prd` → `/gtm` 실행 시: PRD의 타겟 고객/기능을 기반으로 시장 진입 전략 수립
- `/metrics-report` 매월 실행 시: 전월 리포트와 비교해 트렌드 분석

### 결과가 저장되는 폴더

| 업무 영역 | 저장 폴더 | 포함 커맨드 |
|----------|----------|------------|
| 시장조사 | `01-research/` | `/research`, `/competitor`, `/weekly-report` |
| 제품 기획 | `02-product/` | `/prd`, `/lean-canvas`, `/gtm`, `/pricing` |
| 마케팅 | `03-marketing/` | `/seo-audit`, `/campaign`, `/email-sequence`, `/lead-research` |
| 콘텐츠 | `04-content/` | `/blog-post`, `/content-calendar` |
| 서비스 운영 | `10-operations/` | `/metrics-report`, `/release-note`, `/feedback-analysis`, `/incident` |

### CLI 전용 커맨드

`/portfolio-analyze`는 개발 도구가 필요한 커맨드로, Claude Desktop(Cowork)에서는 사용할 수 없습니다. 포트폴리오 분석이 필요하면 개발 환경(Claude Code CLI)에서 실행하세요.
