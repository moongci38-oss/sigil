**작성일**: 2026-02-18

**대상**: AI 지시용 CLAUDE.md와 달리, 사람이 읽는 운영 가이드

## 한눈에 보기

```
business/
│
│ ◆ Track A. 제품 사업 — 돈을 버는 활동
├── 01-research/       시장 파악 (조사, 경쟁사, 트렌드)
├── 02-product/        제품 방향 결정 (전략, 기획, PRD, 로드맵)
├── 03-marketing/      사람들이 알게 만들기 (SEO, 광고, 이메일)
├── 04-content/        콘텐츠 제작 (블로그, 뉴스레터, SNS)
├── 05-design/         화면 & UX (분석, 목업, 에셋)
└── 10-operations/     서비스 운영 (지표, 릴리즈, 장애, 고객지원)
│
│ ◆ Track B. 회사 경영 — 법적·행정 의무 ⛔ AI 접근 제한
├── 06-finance/        재무 & 세무 (계산서, 부가세, 자금)
├── 07-legal/          법무 & 계약 (계약서, 약관, NDA)
└── 08-admin/          경영관리 (4대보험, 프리랜서, 일정)
│
│ ◆ Track C. 시스템 — AI 워크스페이스 도구
└── 09-tools/          AI 도구 관리 (스킬, 에이전트, 스크립트)
```

## Track A. 제품 사업 — 돈을 버는 활동

### 01-research (시장조사)

> **"뭘 만들지 결정하기 전에 시장을 파악하는 곳"**

| 항목 | 내용 |
|------|------|
| **어떤 작업** | 경쟁사 분석, 시장 규모 조사, 고객 리서치, 트렌드 파악 |
| **언제 열어야 하나** | 새 제품 아이디어가 생겼을 때 / 경쟁사 동향이 궁금할 때 / 시장 데이터가 필요할 때 |
| **Claude 요청 예시** | "Notion 경쟁사 분석해줘" / "AI SaaS 시장 트렌드 조사해줘" / "타겟 고객 리서치해줘" |
| **산출물 폴더** | `competitors/`, `trends/`, `market-data/`, `weekly/` |
| **파일명 예시** | `2026-02-18-notion-competitor-analysis.md` |

**활용 팁**: Brave Search MCP로 실시간 웹 검색 + research-engineer 스킬로 체계적 분석

### 06-finance (재무 & 세무) ⛔

> **"돈 들어오고 나가는 것, 세금 납부 기록을 보관하는 곳"**

| 항목 | 내용 |
|------|------|
| **어떤 작업** | 세금계산서 보관, 부가세 신고 준비, 손익계산서, 자금 흐름 관리 |

## 폴더 선택 결정 트리

어느 폴더를 써야 할지 모를 때:

```
새로운 작업이 생겼다
│
├─ "시장/고객/경쟁사 파악이 필요해"
│   └─ → 01-research/
│
├─ "기능이나 제품 방향을 결정해야 해"
│   └─ → 02-product/
│
├─ "사람들에게 알리거나 홍보하고 싶어"
│   └─ → 03-marketing/
│
├─ "글, 포스트, 문서를 써야 해"
│   └─ → 04-content/
│
├─ "화면/UI/UX 관련 작업이야"
│   └─ → 05-design/
│
├─ "서비스 배포 후 운영 문서야"
│   └─ → 10-operations/
│
├─ "돈/세금/법무/계약 관련이야" → B 트랙 (AI 직접 접근 제한)
│   ├─ 재무·세금 → 06-finance/ ⛔
│   ├─ 계약·법무 → 07-legal/ ⛔
│   └─ 행정·총무 → 08-admin/
│
└─ "AI 스킬이나 에이전트 설정이야"
    └─ → 09-tools/
```

## 폴더 간 연결 흐름도

```
[아이디어 발생]
      │
      ▼
 01-research ──────────────────────────────────┐
 (시장 파악)                                    │
      │                                         │
      ▼                                         │
 02-product                                     │
 (전략·기획·PRD)                                 │
      │                                         │
      ├──────────────────────┐                  │
      │                      │                  │
      ▼                      ▼                  │
 05-design              03-marketing ◄──────────┘
 (UI/UX 설계)           (마케팅 전략)
      │                      │
      │               04-content
      │               (콘텐츠 제작)
      │                      │
      ▼                      ▼
[개발: portfolio-project]   [배포·운영]
                                │
                                ▼
                          10-operations
                          (지표·릴리즈·고객지원)
```

**핵심 흐름**:
1. `01-research` → 시장 이해 → `02-product` → 무엇을 만들지 결정
2. `02-product` 결정 후 → `05-design` (어떻게 생겼는지) + `03-marketing` (어떻게 알릴지) **병렬 진행**
3. `03-marketing` 전략 → `04-content` (실제 콘텐츠 생산) **연계**
4. 개발 완료 후 → `10-operations` (운영 단계 돌입)
5. 언제든 → `09-tools` (AI 도구 추가·조정)
