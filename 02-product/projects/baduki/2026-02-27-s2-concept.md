## Executive Summary

로우 바둑이(Low Baduki)는 한국 전통 포커 변형 카드 게임을 현대적 모바일 게임으로 재해석한 프로젝트이다. **로우볼 룰(가장 낮은 핸드 승리)**에 집중하며, **3-5분 세션의 캐주얼 카드 게임**으로, AI 싱글플레이 + 소셜 멀티플레이 + 코스메틱 수집 시스템으로 기존 포커 3사(한게임/피망/윈조이)와 차별화한다. **Unity AI(Assistant, Generators, Inference Engine)**를 개발 전 과정에 적극 활용하여 1인 개발의 한계를 돌파한다. 코스메틱 기반 수익화로 사행성 규제 리스크를 최소화하며, 한국 국내 전용 서비스로 출시한다.

### 1-3. Unique Value Proposition (핵심 가치 제안)

> **"로우 바둑이, 다시 태어나다"** — 한국 전통 카드 게임의 모던 리부트.
> 언제 어디서든 3분 만에 한 판. AI 대전부터 친구 대전까지.

**차별화 4요소**:
1. **로우 바둑이 전문** — 범용 포커가 아닌 로우 바둑이에 올인한 UX + 튜토리얼
2. **AI 대전** — 혼자서도 즐기는 3단계 AI (초보/중수/고수), Unity Inference Engine 온디바이스
3. **현대적 수집** — 카드 스킨, 이모지, 보드 테마 코스메틱 수집
4. **Unity AI 풀 활용** — Generators로 에셋 생성, Assistant로 개발 가속, Inference Engine으로 런타임 AI

### 1-9. Unfair Advantage (불공정 우위)

- **로우 바둑이 전문 특화**: 기존 3사는 범용 포커 → 로우 바둑이에 집중 투자할 동기 없음

- **AI 대전 3단계**: Unity Inference Engine 온디바이스 추론 → 서버 비용 $0
- **1인 개발 + AI 도구**: 저비용 구조 → BEP 낮음 → 생존력 높음
- **WSOP 바둑이 공식 종목** (2023~): 글로벌 인지도 상승 추세

## 5. 경쟁 가설 (Competing Hypotheses)

> S1에서 미비했던 경쟁 가설 3개를 명시적으로 수립.

### 가설 A: "모던 UX 차별화" (포커 경험자 전환)

> 기존 한게임/피망/윈조이 유저 중 UI 불만층을 현대적 로우 바둑이 전용 UX로 전환할 수 있다.

- **타겟**: 기존 포커 게임 활성 유저 중 30-40대
- **핵심 가정**: UI 노후화가 이탈 사유의 상위 3위 안에 포함
- **검증 방법**: 포커 커뮤니티 설문 (N=100+)
- **성공 기준**: 전환 의향 20%+
- **Kill 조건**: 전환 의향 5% 미만

### 가설 B: "AI 싱글플레이로 신규 유저 유입"

> Unity Inference Engine 기반 AI 대전이 로우 바둑이를 처음 접하는 캐주얼 유저의 진입 장벽을 낮춰 시장을 확장할 수 있다.

- **타겟**: 카드게임은 좋아하나 로우 바둑이를 모르는 20-30대
- **핵심 가정**: 튜토리얼 + AI Easy 모드가 로우 바둑이 학습 곡선을 충분히 완화
- **검증 방법**: 프로토타입 유저 테스트 (N=20, 로우 바둑이 미경험자)
- **성공 기준**: 튜토리얼 완주율 80%+, D1 리텐션 30%+
- **Kill 조건**: 튜토리얼 완주율 50% 미만

### 가설 C: "코스메틱 수집이 장기 리텐션 드라이버"

> 코스메틱 수집 시스템(카드 스킨, 시즌 패스)이 D30+ 리텐션의 핵심 드라이버가 될 수 있다.

- **타겟**: 전 세그먼트 (특히 Dolphin 10-15%)
- **핵심 가정**: 로우 바둑이 유저도 수집 욕구가 있으며, Unity AI Generators로 생성한 코스메틱에 기꺼이 지불
- **검증 방법**: 소프트 런치 A/B 테스트 (수집 있음 vs 없음)
- **성공 기준**: 수집 그룹 D30 리텐션 +5%p 이상
- **Kill 조건**: 수집 유무 간 리텐션 차이 2%p 미만

### 가설 우선순위

**가설 B > A > C** (검증 순서)
- B는 시장 확장의 핵심 → 프로토타입 단계에서 가장 먼저 검증
- A는 초기 유저 확보 → 소프트 런치 전 커뮤니티 검증
- C는 장기 수익화 → 소프트 런치 A/B 테스트

## 8. Kill Criteria 점검

| 기준 | 상태 | 상세 |
|------|:----:|------|
| TAM < $1M | ✅ 해당 없음 | TAM $7.5B → 충분 |
| 경쟁사 70%+ 점유 | ⚠️ 주의 | 한게임/피망/윈조이 3사 80%+ 추정, 단 로우 바둑이 전문 경쟁자는 0개 |
| 핵심 기술 불가 | ✅ 해당 없음 | Unity + Firebase + Sentis 모두 검증됨 |
| 규제 장벽 | ✅ 해당 없음 | GRAC 12세 이용가 통과 가능 |

**경쟁사 점유율 해석**: 3사가 "범용 포커" 시장을 80% 점유하나, "로우 바둑이 전문" 카테고리에서는 경쟁자 0개. 신규 카테고리 창출로 Kill Criteria 회피 가능.

**최종 판정**: Kill Criteria **해당 없음** → 진행 가능

## 12. Unity AI 활용 전략

> 참조: https://unity.com/kr/features/ai

Unity AI 3대 기능을 개발 전 과정에 적극 활용하여 1인 개발의 한계를 돌파한다.

### 12-1. Unity AI Assistant (개발 생산성)

| 활용 영역 | 적용 |
|----------|------|
| 코드 생성 | 게임 로직, 네트워크 코드, UI 바인딩 등 사전 컴파일된 코드 생성 |
| 에이전트 작업 | 콘솔 오류 자동 해결, 파일명 일괄 변경, NPC 배리언트 생성 |
| 씬 구성 | 로비/테이블/결과 화면 등 씬 오브젝트 자동 배치 |
| 디버깅 | 에셋 드래그앤드롭 컨텍스트 기반 문제 해결 |

### 12-2. Unity AI Generators (에셋 생성)

| 에셋 유형 | 활용 |
|----------|------|
| 스프라이트 | 카드 덱, 칩/토큰, 아이콘, 캐릭터 아바타 생성 |
| 텍스처 | 테이블 배경, UI 패턴, 카드 뒷면 디자인 |
| 애니메이션 | 카드 뒤집기/배분/교환, 승리/패배 이펙트 |
| 사운드 | 카드 효과음, 배경 음악, UI 사운드 |
| 후처리 | 배경 제거, 업스케일링, 픽셀화, 색상 조정 |

### 12-3. Unity Inference Engine (런타임 AI)

| 활용 | 설명 |
|------|------|
| AI 대전 상대 | 온디바이스 추론으로 AI 상대 구현 (서버 의존 없음, 비용 $0) |
| 난이도 조절 | 로컬 모델로 실시간 난이도 동적 조정 (Easy/Medium/Hard) |
| 패턴 분석 | 플레이어 행동 패턴 분석 → 맞춤형 튜토리얼/힌트 |

### 12-4. Unity AI vs 외부 AI 역할 분담

| 영역 | Unity AI | 외부 AI (Stitch, NanoBanana) |
|------|----------|------------------------------|
| 코드/개발 | Assistant (코드 생성, 디버깅) | -- |
| 2D 에셋 | Generators (스프라이트, 텍스처, 사운드) | 고품질 일러스트, 키비주얼 |
| 런타임 AI | Inference Engine (온디바이스) | -- |
| AI 대전 | Inference Engine + CFR/규칙 기반 | -- |
